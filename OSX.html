Colonyzer: installation under OSX

I don't have regular access to OSX machines, but Katja Luck and Adriana San Miguel have kindly provided the notes below which outline successful installation on various flavours of OSX.

  1. Install python 2.7 from python.org and make sure it's the python version used by the system
  2. Install the most recent version of Xcode for your operating system (needed by homebrew)
    • Updating current installation is not possible
    • Delete current Xcode installation from applications folder by simply trashing the Xcode app
    • Install new Xcode
    • With the newer versions of Xcode, install Xcode command line tools via preferences/download. With older versions of Xcode, this is given as an option during Xcode installation
  3. Install homebrew
    • type into terminal: 
      ruby -e "$(curl -fsSL https://raw.github.com/mxcl/homebrew/go)"
    • different procedure for Mac OS X 10.5, see webpage http://brew.sh/
    • type in terminal: 
      brew doctor
    • WARNING: brew may complain about order of paths
  4. Change path in .bash_profile for homebrew
    • http://apple.stackexchange.com/questions/53057/homebrew-path-error-mac-os-x-10-6-8
    • Look in your home directory ('cd ~' to get there, 'ls -a' to look) for a .bash_profile or .bashrc file.
    • If you find one of them, open it up and you should see a line like this: 
      export PATH=/usr/bin:/usr/local/bin:/opt/local/sbin:$PATH
    • Cut the "/usr/bin:" and paste it after the "/usr/local/bin:"
    • If you don't find either of those files, create one called .bash_profile and put this line inside: 
      export PATH=/usr/local/bin:$
  5. installation of gfortran compiler with homebrew (needed for scipy):
    • type into terminal: 
      brew install gfortran
  6. installation of libjpeg library with homebrew (needed for PIL):
    • type into terminal: 
      brew install libjpeg
  7. installation of pip package manager and the dependency:
    • https://pypi.python.org/pypi/pip
    • http://www.pip-installer.org/en/latest/installing.html
    • downloaded the scripts ez_setup.py and get-pip.py for installing the ez_setup dependency for pip and pip itself
  8. installation of numpy, scipy, PIL, pandas, matplotlib with pip:
    • type in terminal: 
      pip install package_name
  9. for installation of pygame:
    • http://jalada.co.uk/2011/06/17/installing-pygame-on-os-x-with-a-homebrew-python-2-7-install.html
    • install some dependencies first with homebrew - type into terminal: 
      brew install sdl sdl_ttf sdl_image sdl_mixer
    • for a reason I can't remember the following instructions on that page did not work for us, so we installed the .dmg file from pygame.org for python 2.7 and Mac
  10. install colonyzer with pip
    • type into terminal: 
      pip install Colonyzer2
    • this brought some warnings about a scripts.py file that it couldn't locate but still said that installation was successful
    • for running colonyzer we had to do the following trick:
      • first, run parametryzer as follows in terminal in folder that contains the image: 
        python2.7-32 path_to_parametryzer/parametryzer
      • run independent as usual: 
        independent

Tips for installing colonyzer on Mac OS X 10.5.8:

  • brew could not install the gfortran compiler
    • I downloaded the installer directly from the web
  • pip could not install matplotlib but using an installer from the web worked out
  • for this Mac OS X version only a python 2.7. 32 bit version was available from python.org
    • this led to the fact that parametryzer runs again without the extra specification of python2.7-32

    • Tips for installing colonyzer on Max OS X 10.8.4:

    • installation of matplotlib required prior installation of freetype: 
      brew install freetype
    • web page with detailed infos: www.tapir.caltech.edu/~dtsang/python.html

    In general, I think that the most important thing in getting colonyzer running is to have python 2.7 installed from python.org. It automatically will use this python version as the current python version once it is installed, which is very helpful.

    The pip package manager then, too, uses this python version and my impression is that when installing pygame with a .dmg file, it also finds this python version.

    I can only warn people not to use MacPorts as package manager to install the colonyzer dependencies because I think this uses the mac-intern python version.
